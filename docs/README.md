# Documentation Map

This repository separates documentation by intent:

- `docs/architecture/` — architecture and system-structure docs.
- `docs/product/` — product framing and positioning docs.
- `docs/benchmark/` — benchmark specification, schema, and scoring documentation.
- `docs/research/` — research/supporting literature notes (not active SPIN bundles).
- `docs/migration/` — migration audits, cleanup reports, and restructuring plans.
- `docs/archive/` — historical/legacy artifacts retained for traceability.

Operational datasets and runtime assets should live outside `docs/` (e.g., `data/`, `flow-db/`, `corpora/`).
